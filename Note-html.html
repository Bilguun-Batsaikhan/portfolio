HTML and Web Fundamentals

How Internet works?

Whenever a client makes a request over the internet using a specific URL link address, it makes the request using the HTTP protocol. The server then sends back a response, which can include an HTML file that is subsequently rendered on the browser. This HTML file can also contain links to other assets such as JavaScript, CSS, or images, for which the browser makes subsequent requests to the server.

Each client machine has its own IP address, but the public internet will typically see only one gateway address. For instance, if multiple devices are connected to the same internet connection, the router’s IP address will be visible to the public rather than each individual device’s address.

Conversely, an internet domain address is converted to a real IP address through DNS (Domain Name System). You can imagine DNS as a service that maintains a record of all domain names and their corresponding IP addresses.

HTTP

HTTP stands for Hypertext Transfer Protocol and serves as a communication language between client and server.

  • HTTP is stateless, meaning each request/response cycle is an independent transaction.
  • HTTP responses include status codes such as 200 (OK) and 404 (Not Found).
  • The most common HTTP request methods are CRUD operations: GET, POST, PUT, DELETE.

HTML Common Tags

<p> and <h> are block-level tags, meaning they will take up the full available width and start on a new line.

Lists

To create a list, first use <ul> (unordered list) or <ol> (ordered list), then nest them with <li> (list item) tags. 

              
<ul>
  <li>Item 1</li>
  <li>Item 2</li>
  <li>Item 3</li>
  <li>Item 4</li>
</ul>

Browser will render these tags with default styling with margin and padding.

Input Field Attributes

<form>
          <div>
            <label for="name">Name</label>
            <input
              type="text"
              id="name"
              placeholder="Enter name"
              name="name"
              required
              minlength="3"
              maxlength="10"
            />
          </div>
          ...
        </form>
Attribute / Tag Purpose Explanation
<form> Form container Groups input fields for data submission.
<div> Container Used for layout and grouping elements.
<label for="name"> Input label Describes the input field; linked via for="name".
for="name" Link label to input Matches the id of the input so clicking the label focuses the input.
<input> User input field Accepts data from the user.
type="text" Input type Creates a single-line text input.
id="name" Identifier Uniquely identifies the element; used by label and JavaScript.
placeholder="Enter name" Hint text Shows a suggestion inside the input box before the user types.
name="name" Form data key Used as the key when the form is submitted (e.g., name=John).
required Field must be filled Prevents form submission if the input is empty.
minlength="3" Minimum characters Requires at least 3 characters to submit.
maxlength="10" Maximum characters Limits input to 10 characters max.

Select Boxes & Text Area

<form>
          <label for="product">Product</label>
          <br />
          <select name="product" id="product" multiple size="4">
            <option value="iphone">iPhone</option>
            <option value="imac" selected>iMac</option>
            <option value="macbook">Macbook</option>
            <option value="macbook-pro">Macbook Pro</option>
          </select>
        </form>

You can also use shortcut select>option*5 to generate 5 options quickly.

If you want a longer input then use textarea. You can control the width with cols, height with rows attributes. 

<textarea
          name="message"
          id="message"
          placeholder="Enter your issue"
          rows="4"
          cols="40"
        ></textarea>

Tables

Tables are for tabular data, don’t ever use it for layout!

First Name Last Name Email Telephone Numbers

<table>
          <colgroup>

            <col style="background-color: darkcyan" />
